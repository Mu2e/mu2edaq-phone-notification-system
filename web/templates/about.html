About

Mu2e DAQ Notify (v{{ version }}) collects error and warning events from Mu2e DAQ applications, filters them, and pushes notifications to registered iPhones (via Apple Push Notification service), Slack, and Discord. Events arrive over an HTTP JSON API or the DAQ's existing ZeroMQ publishing scheme, and the server announces itself on the DAQ network with the mu2edaq-discovery protocol.

Server dependencies

  • Flask — web framework (BSD-3)
  • SQLAlchemy — database layer, SQLite by default with Postgres hooks (MIT)
  • PyYAML — YAML configuration (MIT)
  • httpx with HTTP/2 — APNs and webhook delivery (BSD-3)
  • PyJWT + cryptography — APNs provider tokens and enrollment tokens
  • segno — QR code generation (BSD-3)
  • Authlib — Fermilab SSO / OpenID Connect (BSD-3, optional)
  • pyzmq — ZeroMQ compatibility listener (BSD-3, optional)

Web interface dependencies

  • Tailwind CSS (CDN) — styling
  • htmx (CDN) — partial page updates
  • Server-Sent Events — live dashboard push (no polling)

Companion components

  • mu2edaq_notify Python publisher library (stdlib only) and mu2edaq-notify CLI
  • mu2edaq-notify-cpp C++17 publisher library (libcurl)
  • Mu2e Notify iOS application (SwiftUI)
  • mu2edaq-discovery — service auto-discovery
